Broken version
##############
(Fit_const, Fit_const.const_weight) ==
    GET /INPUT/  [Fit.consts, Const.const_name |AS| 'name']
        /FUNC/   SimpleFunc(parse_dict, ['consts','name'],...)

Fixed
#####

(Fit_const, Fit_const.const_weight) ==
    GET /INPUT/  [Fit.consts, Const.const_name |AS| 'cname']
        /FUNC/   SimpleFunc(parse_dict,['consts','cname'],...)

The problem was that "name" is an identifying attribute of Fit, so it got
overwritten and the hash of the resulting row in the mapping table ignored
the fit.name attribute. This led to the same rows getting overwritten.

Not a 'bug', but it is extremely easy for users to shoot themselves in the foot
with horrible buglike results.
